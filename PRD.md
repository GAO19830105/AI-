## AI 海龟汤游戏网站 PRD

### 1. 产品概要
- **产品名称**：AI 海龟汤推理游戏
- **产品形态**：Web 网站（桌面端优先，自适应移动端）
- **一句话描述**：一个由 AI 主持的在线海龟汤推理游戏，玩家通过与 AI 对话，提问“是/否/无关”问题，逐步推理出故事真相。

---

### 2. 目标与价值
- **核心目标**
  - 提供一个轻量、上手快的推理小游戏网站，核心乐趣来自“不断提问 + 推理还原真相”。
  - 利用 AI 自动生成和主持海龟汤，减少人工维护题库成本。
  - 通过良好的对话体验与悬疑氛围，提升用户停留时长与回访率。
- **业务价值**
  - 作为 AI 应用 Demo/作品集，展示 AI 在互动娱乐/游戏主持场景中的应用。
  - 为后续增加用户系统、排行榜、多 人模式等留出扩展空间。

---

### 3. 目标用户
- **喜欢推理游戏的玩家**
  - 特征：喜欢“狼人杀”“密室逃脱”“剧本杀”等逻辑推理类产品。
  - 需求：有趣的故事设定、可反复游玩的关卡、适中的难度梯度。
- **想要轻松娱乐的上班族**
  - 特征：碎片时间娱乐，偏好 5–15 分钟一局的轻度游戏。
  - 需求：无需注册即可开玩、简单明了的操作、节奏可控。
- **对 AI 应用感兴趣的技术爱好者**
  - 特征：关注大模型应用场景，对 AI 担任主持人感兴趣。
  - 需求：看到 AI 的“智商”“主持风格”，对交互体验和稳定性敏感。

---

### 4. 使用场景与用户故事
- **场景 1：午休碎片娱乐**
  - 作为一个上班族，在午休 15 分钟时，我可以打开网站随便点一个海龟汤故事，轻松玩一局，不需要注册登录。
- **场景 2：AI 推理体验**
  - 作为一个 AI 爱好者，我想体验 AI 当主持人、我来提问的海龟汤游戏，感受 AI 如何根据固定汤底约束自己回答“是/否/无关”。
- **场景 3：讲故事给同伴听**
  - 作为一个喜欢社交的玩家，我希望在玩完一局后，可以查看完整汤底，并把这个故事讲给朋友玩（线下主持），所以需要清晰的汤面+汤底呈现。

---

### 5. 范围与版本规划
#### 5.1 MVP 范围（本 PRD 重点）
1. **游戏大厅**
   - 显示一组可供选择的海龟汤故事（列表/卡片）。
   - 支持显示故事标题、难度标签、类型标签（例如：悬疑/温情/脑洞）。
   - 点击任意故事进入游戏页面。
2. **游戏页面**
   - 上方区域展示**汤面故事**（只读）。
   - 中间为**对话区**：玩家提问与 AI 回答以气泡形式显示，类似微信对话。
   - 底部为**输入框**：玩家输入任意自然语言问题，点击发送。
   - AI 主持人只返回“是/否/无关”三类含义的回答（文本上可以是“是的/不是/这个无关”等）。
   - 显示完整的**对话历史**（本局内所有问答）。
   - 提供明显的“**查看汤底**”按钮（仅在结束后可点，或按策略控制）。
   - 提供“**结束游戏**”按钮（结束当前局）。
3. **汤底页面**
   - 展示当前游戏的**完整汤底（真相）**，包括清晰的事件/人物/关键事实描述。
   - 展示玩家本局的**推理过程**概览（对话历史概览，或关键问题列表）。
   - 提供“**再来一局**”按钮（回到大厅或直接开新局）。

#### 5.2 后续版本（不在本次必须范围）
- 用户登录系统
- 游戏记录长期保存与回放
- 排行榜（如通关次数、平均发问数等）
- 用户自定义海龟汤（上传/编辑汤面和汤底）
- 多人对战模式（多人共用一个房间，一起提问）

### 6. 详细功能说明（MVP）

#### 6.1 游戏大厅
- **入口**
  - 网站首页即为游戏大厅。
- **信息展示**
  - 每个海龟汤以卡片展示，包含：
    - 标题（必填）
    - 难度标签：`简单 / 普通 / 困难`
    - 类型标签：如 `悬疑 / 黑色幽默 / 温情 / 反转`
    - 简短一句话简介（不剧透）
  - 数量可以是：
    - 固定题库（初期可手工配置几道经典海龟汤，后期支持 AI 生成）。
- **交互**
  - 点击卡片：
    - 创建一局新的游戏（game session），带上选择的故事 ID。
    - 跳转到游戏页面，URL 可带 `gameId` 和 `storyId` 参数。

#### 6.2 游戏页面
- **布局结构**
  - 顶部：汤面故事卡片（可折叠/展开）。
  - 中部：聊天区，类似微信对话形式：
    - 左侧为 AI 主持人的气泡。
    - 右侧为玩家提问的气泡。
  - 底部：文本输入框 + 发送按钮。
  - 页面某处（顶部右侧或底部工具栏）放置：
    - “结束游戏”按钮
    - “查看汤底”按钮（未结束时置灰或隐藏）

- **汤面展示**
  - 内容：当前选择故事的汤面文本。
  - 风格：深蓝背景卡片，白色/浅色文字，营造悬疑氛围。

- **提问与回答逻辑**
  - 玩家输入任意文字提问（自然语言）。
  - 前端将问题发送到后端接口 `/api/games/:id/ask`。
  - 后端基于：
    - 固定汤底（结构化存储）
    - 历史对话
    - 规则/AI 模型
    - 返回三类判定之一：**是 / 否 / 无关**（必要时也可支持“不确定”，但 UI 首期只展示三类）。
  - 前端在聊天窗口中显示：
    - 右侧：玩家提问。
    - 左侧：AI 的回答文本（如“是的”“不是”“这个问题对真相无关紧要”）。

- **对话历史**
  - 当前局内的所有问答从上至下按时间排序显示。
  - 页面刷新后（在同一局内）能恢复本局历史（需要后端以 `gameId` 维度存储）。

- **结束游戏**
  - 点击“结束游戏”：
    - 弹出确认对话框：`确认结束本局游戏吗？结束后可查看汤底。`
    - 确认后：
      - 后端更新本局状态为 `ended`。
      - 页面跳转到汤底页面（或出现“查看汤底”按钮可点击）。

- **查看汤底**
  - 仅在游戏状态为 `ended` 时可点。
  - 点击后跳转到汤底页面，携带 `gameId`。

#### 6.3 汤底页面
- **汤底展示**
  - 清晰展示本故事的完整真相：
    - 发生了什么、人物关系、关键反转点。
  - 建议采用分块结构：
    - 故事总结（1–3 段）
    - 时间线（按时间顺序列出关键事件）
    - 关键事实列表
  - 视觉风格：有“揭晓真相”的仪式感，例如：
    - 打开式动效（淡入、遮罩拉开等）
    - 深色背景 + 高亮标题

- **玩家推理过程**
  - 列表展示本局的对话简史：
    - 可以完整展示问答记录
    - 或展示精简版本：例如标出“命中关键点的问题”
  - 目的：让玩家回顾自己是如何一步步接近真相。

- **再来一局**
  - 提供按钮“再来一局”：
    - 点击后返回游戏大厅。
    - 可选：自动随机选一则新故事并直接进入游戏。

### 7. 界面与视觉要求

#### 7.1 整体风格
- 主题：**神秘悬疑，深蓝色调**。
- 主色：深海蓝、墨蓝等；辅以少量高亮色（如荧光蓝/紫）用于按钮与重点信息。
- 字体：简洁易读，中文推荐常用 Web 安全字体或系统字体组合。

#### 7.2 游戏大厅
- 布局：卡片式网格布局（2–3 列视口自适应）。
- 卡片内容：
  - 标题
  - 难度标签（颜色区分）
  - 类型标签（小标签）
  - 简要描述（1–2 行）
- 悬停效果：卡片有轻微放大/光晕效果，提升“可点击性”。

#### 7.3 游戏页面
- 聊天 UI：参考微信/主流 IM：
  - AI 气泡：靠左，深色气泡，附带“AI 主持人”的头像（可用简约图标）。
  - 玩家气泡：靠右，稍亮色/高对比度气泡。
  - 时间分隔可以简单处理（可选）。
- 汤面卡片：
  - 置顶固定，深蓝背景，略带噪点/纹理以增加神秘感。
  - 支持折叠（展开/收起）。

#### 7.4 汤底页面
- 视觉主题：揭晓真相，适度“戏剧化”。
- 动效建议：
  - 页面进入时汤底卡片从模糊/暗色逐渐清晰亮起。
  - 关键事实依次淡入。

### 8. 技术方案（概要）

#### 8.1 前端
- **技术栈**：React + TypeScript + Tailwind CSS
- **主要页面**
  - `Hall`（游戏大厅）
  - `Game`（游戏页面）
  - `Solution`（汤底页面）
- **状态管理**
  - 最小化全局状态；每个页面通过 URL 参数获取 `gameId`。
  - 对话历史通过 API 获取并存在前端状态中，必要时可做简单缓存。

#### 8.2 后端
- **技术栈**：Node.js + Express
- **职责**
  - 提供游戏列表（故事元数据）。
  - 创建新游戏局（生成/加载汤面与汤底）。
  - 接收玩家提问，请求 AI 接口，根据汤底判定并返回答案。
  - 存储对话历史与游戏状态（可先内存/简单存储，后续接 DB）。

#### 8.3 AI 模块
- **使用服务**：DeepSeek / Claude API（二选一或可配置）。
- **策略**
  - MVP 阶段可采用“固定题库 + AI 判定”：
    - 汤底由人工预设在服务端。
    - AI 接口的 Prompt 中提供完整汤底 + 汤面 + 对话历史概要，让模型仅输出“是 / 否 / 无关”。
  - 后续可以扩展为“AI 生成新海龟汤”。

#### 8.4 部署
- **前端**：部署在 Vercel（React + Next.js 或纯前端 SPA）。
- **后端**：
  - 可部署在 Vercel 的 Serverless Functions 或其他 Node.js 运行环境。
  - 需配置环境变量用于存储 AI API Key 等敏感信息。

### 9. 数据与接口（MVP 粗略定义）

#### 9.1 游戏列表
- `GET /api/stories`
- 响应：
```json
[
  {
    "id": "story_1",
    "title": "深夜的电话",
    "difficulty": "medium",
    "category": "suspense",
    "summary": "一个男人接到了一通奇怪的电话……"
  }
]
```

#### 9.2 创建新游戏
- `POST /api/games`
- 请求：
```json
{
  "storyId": "story_1"
}
```
- 响应：
```json
{
  "gameId": "game_123",
  "surfaceStory": "你只知道：在一个深夜……",
  "status": "active"
}
```

#### 9.3 提问
- `POST /api/games/:gameId/ask`
- 请求：
```json
{
  "question": "他是在家里死的吗？"
}
```
- 响应：
```json
{
  "answer": "是的。",
  "label": "YES", 
  "createdAt": "2026-03-19T00:00:00Z"
}
```

#### 9.4 获取对话历史
- `GET /api/games/:gameId/messages`
- 响应：
```json
[
  {
    "role": "user",
    "content": "他是在家里死的吗？",
    "createdAt": "2026-03-19T00:00:00Z"
  },
  {
    "role": "assistant",
    "content": "是的。",
    "label": "YES",
    "createdAt": "2026-03-19T00:00:01Z"
  }
]
```

#### 9.5 结束游戏
- `POST /api/games/:gameId/end`
- 响应：
```json
{
  "status": "ended"
}
```

#### 9.6 获取汤底
- `GET /api/games/:gameId/solution`
- 响应：
```json
{
  "summary": "完整故事描述……",
  "timeline": [
    "几小时前，他接到了一通电话……",
    "实际上那通电话来自……"
  ],
  "keyFacts": [
    "事实1……",
    "事实2……"
  ]
}
```

### 10. 关键非功能需求
- **性能**
  - 普通页面加载时间 < 3 秒（正常网络环境）。
  - AI 回答时间目标 < 5 秒（取决于模型延迟）。
- **可用性**
  - 无需登录即可体验完整单局游戏流程。
  - 页面刷新后能恢复当前局的对话（依赖 `gameId` 与后端存储）。
- **稳定性**
  - AI 接口失败时，给出友好提示并支持重试。
  - 后端需对异常输入做基本校验（空问题等）。
- **安全性**
  - AI 提示词需控制内容安全，避免生成明显不适宜内容。

### 11. 验收标准（针对 MVP）
1. **游戏大厅**
   - 打开网站能够看到至少 3 个不同故事卡片，包含标题、难度和简介。
   - 点击任意卡片可以进入游戏页面，并展示对应的汤面。
2. **游戏页面**
   - 玩家可以输入自然语言问题，AI 返回“是/否/无关”语义的回答。
   - 对话在同一局内完整记录并展示在聊天区。
   - 刷新页面后仍可以恢复当前游戏局的对话历史。
   - 点击“结束游戏”后，当前游戏状态变更为已结束。
3. **汤底页面**
   - 结束游戏后，可以查看该故事的完整汤底。
   - 页面显示本局玩家的提问记录（完整或摘要）。
   - 可以通过“再来一局”返回大厅并开始新故事。
4. **整体体验**
   - 视觉风格符合“神秘悬疑、深蓝色调”的要求。
   - 从“进入网站 → 选故事 → 提问对话 → 结束 → 看汤底 → 再来一局”全链路无明显阻塞或致命错误。
