# AGENTS.md — AI 海龟汤游戏

本文件指导 Cursor / AI 助手在本项目中的开发行为。自动编写或修改的代码应遵循以下约定。

---

## 1. 项目概述

- **产品**：AI 海龟汤游戏网站（玩家提问，AI 按规则回答）。
- **技术栈**：React + TypeScript + Tailwind CSS。
- **实现方式**：函数式组件 + Hooks；样式仅用 Tailwind（与现有工程约定一致时可配合 Vite 等构建工具）。

**本地运行**：

- **前端**：`npm run dev`（仓库根目录；Vite，默认 http://localhost:5173）。
- **后端**：`node server.js`（在 `server` 目录执行；默认 http://localhost:3000；依赖与 `.env` 见该目录）。

---

## 2. 开发规范

| 要求 | 说明 |
|------|------|
| TypeScript | 业务代码全面使用 TS，保证类型安全。 |
| 组件范式 | 仅使用函数式组件与 Hooks，不使用类组件。 |
| 样式 | 使用 Tailwind CSS 编写样式。 |
| 组件设计 | 组件可复用；对关键逻辑与非显而易见的设计加简短注释（避免对显而易见代码堆砌注释）。 |
| 复杂度 | 保持简洁，避免过度设计；优先实现核心玩法与关键路径。 |

---

## 3. 代码风格

- **组件名**：PascalCase（例：`GameBoard`, `ChatPanel`）。
- **函数名**：camelCase（例：`submitQuestion`, `loadStory`）。
- **常量**：UPPER_SNAKE_CASE（例：`MAX_QUESTIONS`, `DEFAULT_MODEL`）。
- **类型名**：以 `T` 开头（例：`TStory`, `TMessage`, `TGamePhase`）。

**目录参考**（扩展时与现有仓库保持一致）：

- `src/components/` — 可复用 UI
- `src/pages/` — 页面
- `src/api/` — AI / HTTP 封装（敏感逻辑集中在此）
- `src/context/`、`src/hooks/` — 状态与逻辑复用（按需）

---

## 4. 设计要求

- **整体**：神秘、悬疑感；主背景以深蓝色调为主，如 `bg-slate-900`（可与 `slate-950` 等搭配渐变）。
- **强调色**：金色系，如 `text-amber-400`，以及配套的 `border-amber-400`、`ring-amber-400` 等。
- **圆角**：统一 `rounded-lg`（特殊元素需说明理由再偏离）。
- **阴影**：统一 `shadow-lg`；交互态可升级到 `shadow-xl` 等，保持风格一致。
- **响应式**：必须考虑移动端；使用 Tailwind 断点（`sm:` / `md:` / `lg:`）避免布局塌陷与横向溢出。

---

## 5. 安全与配置

- **禁止**在源码、配置提交物中硬编码真实 AI API Key 或密钥。
- 通过**环境变量**注入（例如 Vite 下 `VITE_*` 前缀变量，或由后端代理时仅服务端读取密钥）。
- 示例与文档中使用占位符（如 `YOUR_API_KEY`），并注明由环境变量提供。
- AI 调用应封装在独立模块（如 `src/api/`），页面/组件只调用封装函数，不散落裸请求。

---

## 6. 测试要求

- **手动测试**：每完成一个功能，应能按步骤自测（入口、正常/边界输入、预期结果）。
- **多端**：在常见视口宽度下检查布局（手机宽度、平板、桌面）。
- **AI 行为**：用典型汤面/汤底用例验证回答是否符合规则（例如是否/无关等约定）；必要时提供 mock 客户端以便无 Key 或离线调试 UI。
